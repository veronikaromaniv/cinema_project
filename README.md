# 🎬 Cinema Project: Django ORM & PostgreSQL

Навчальний проєкт для побудови системи управління каталогом фільмів.

---

## 🚀 Технологічний стек

* **Framework:** Django 5.x
* **Database:** PostgreSQL
* **Language:** Python 3.10+
* **Libraries:**

  * `psycopg2-binary`
  * `python-dotenv`

---

## 🧱 Моделі даних

Проєкт базується на чотирьох пов'язаних моделях:

1. **Genre** — назва жанру.
2. **Director** — інформація про режисера:

   * ім'я
   * країна
   * рік народження
3. **Movie** — дані про фільм:

   * назва
   * рейтинг
   * тривалість
   * рік
   * `ForeignKey` на Genre та Director
4. **Review** — відгуки:

   * оцінка
   * текст
   * зв'язок із фільмом

---

## ⚙️ Налаштування та запуск

### 1. 📦 Створення бази даних

```sql
CREATE DATABASE cinema_db;
```

---

### 2. 📥 Встановлення залежностей

```bash
pip install django psycopg2-binary python-dotenv
```

---

### 3. 🔐 Конфігурація середовища

Створіть файл `.env` у кореневій папці:

```env
DB_NAME=cinema_db
DB_USER=postgres
DB_PASSWORD=ваш_пароль
DB_HOST=localhost
DB_PORT=5432
SECRET_KEY=ваша_секретна_строка
```

---

### 4. 🧬 Міграції

```bash
python manage.py migrate
```

---

### 5. 🌱 Наповнення бази даними

Запустіть скрипт для створення тестових даних:

```bash
python manage.py shell < populate_db.py
```

Скрипт створює:

* 22 фільми
* 6 режисерів
* відгуки

---

### 6. ▶️ Запуск серверу

```bash
python manage.py runserver
```

---

## 🧠 ORM-запити

Основна логіка знаходиться у файлі `queries.py`.

Реалізовано 15 завдань, що покривають:

### 🔎 Фільтрація

* `filter()`
* `exclude()`
* `Q`-об'єкти (складні умови AND/OR)

### 📊 Агрегації та анотації

* `Avg` — середній рейтинг
* `Count` — кількість фільмів
* `Min` / `Max` — роки

### ⚡ Оптимізація

* `update()` — масове оновлення
* `F()` — операції на рівні БД без завантаження в пам'ять

### 🔗 Робота зі зв'язками

* зворотні зв'язки (`review_set`)
* фільтрація через пов'язані таблиці

### ▶️ Запуск перевірки

```bash
python queries.py
```
